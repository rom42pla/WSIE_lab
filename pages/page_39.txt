       <div class="section" id="abstract"> 
        <h1><a class="toc-backref" href="#id2">Abstract</a></h1> 
        <p>At present, setting the <tt class="docutils literal">PYTHONPATH</tt> environment variable is the only method for defining additional Python module search directories.</p> 
        <p>This PEP introduces the '-P' valued option to the python command as an alternative to <tt class="docutils literal">PYTHONPATH</tt>.</p> 
       </div> 
       <div class="section" id="rationale"> 
        <h1><a class="toc-backref" href="#id3">Rationale</a></h1> 
        <p>On Unix:</p> 
        <pre class="literal-block">python -P $SOMEVALUE
</pre> 
        <p>will be equivalent to:</p> 
        <pre class="literal-block">env PYTHONPATH=$SOMEVALUE python
</pre> 
        <p>On Windows 2K:</p> 
        <pre class="literal-block">python -P %SOMEVALUE%
</pre> 
        <p>will (almost) be equivalent to:</p> 
        <pre class="literal-block">set __PYTHONPATH=%PYTHONPATH% &amp;&amp; set PYTHONPATH=%SOMEVALUE%\
   &amp;&amp; python &amp;&amp; set PYTHONPATH=%__PYTHONPATH%
</pre> 
       </div> 
       <div class="section" id="other-information"> 
        <h1><a class="toc-backref" href="#id4">Other Information</a></h1> 
        <p>This option is equivalent to the 'java -classpath' option.</p> 
       </div> 
       <div class="section" id="when-to-use-this-option"> 
        <h1><a class="toc-backref" href="#id5">When to use this option</a></h1> 
        <p>This option is intended to ease and make more robust the use of Python in test or build scripts, for instance.</p> 
       </div>

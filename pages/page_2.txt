            <h1><a name="Sistemi_distribuiti"></a> Sistemi distribuiti </h1> 
            <p></p> Docente: 
            <a href="http://www.dsi.uniroma1.it/~mei" target="_blank">Alessandro Mei<img alt="" height="12" border="0" width="13" src="/pub/TWiki/TWikiDocGraphics/external-link.gif"></a> 
            <p></p> 
            <h2><a name="Avvisi"></a> Avvisi </h2> 
            <p></p> 
            <ul> 
             <li> Questo sito web è vecchio. Il sito per l'anno accademico 2011-12 è <a href="https://sites.google.com/site/sistemidistribuitimeisapienza/" target="_blank">https://sites.google.com/site/sistemidistribuitimeisapienza/<img alt="" height="12" border="0" width="13" src="/pub/TWiki/TWikiDocGraphics/external-link.gif"></a>. </li>
            </ul> 
            <p></p> 
            <h2><a name="Obiettivi"></a> Obiettivi </h2> 
            <p></p> In questo corso tratteremo alcune delle idee fondamentali usate per costruire i moderni sistemi distribuiti. Vedremo anche alcune tecniche che saranno usate nei sistemi futuri, almeno crediamo. Ci occuperemo dei fondamenti: problemi, protocolli, algoritmi e limiti insormontabili. Vedremo, anche nei dettagli, le idee usate nella costruzione dei sistemi peer-to-peer. Non ci occuperemo di CORBA, RMI, HTTP, RPC, eccetera (ci sono altri ottimi corsi per questo). Alcuni degli argomenti sono in qualche modo classici, li troveremo nei libri. Molti altri in articoli scientifici. 
            <p></p> 
            <h2><a name="Prerequisiti"></a> Prerequisiti </h2> 
            <p></p> Quello che mi aspetto è che abbiate nel vostro background i concetti fondamentali dei sistemi operativi, come dovrebbe avere qualsiasi laureato triennale in Informatica o Ingegneria Informatica. Non c'e' nessuna propedeuticità formale. 
            <p></p>

            <h2><a name="Esame"></a> Esame </h2> 
            <p></p> L'esame consiste in una prova scritta a metà corso e una a fine corso. 
            <p></p> 
            <font color="red">Attenzione!</font> C'è stato un imprevisto con l'organizzazione di un importante convegno in cui il dipartimento è coinvolto. La consequenza è che nella settimana dal 19 al 23 aprile non ci sarà lezione (tutte le aule in via Salaria sono occupate dal convegno!), che la settimana seguente non ci saranno esoneri ma lezione, e che il nostro esonero è stato spostato a 
            <strong>martedì 20 aprile ore 12.00, aula alfa</strong>. 
            <p></p> 
            <h3><a name="Primo_Esonero"></a> Primo Esonero </h3> 
            <p></p> L'esonero si terrà 
            <strong>martedì 20 aprile ore 12.00, aula alfa</strong>. Preparatevi studiando la teoria e facendo tutti gli esercizi del diario delle lezioni. Ricordatevi delle seguenti cose importanti: 
            <p></p> 
            <ol> 
             <li> è <strong>obbligatorio prenotarsi su Infostud</strong> all'appello estivo degli esoneri entro il 19 aprile; </li> 
             <li> all'esonero non è possibile portare libri o appunti. </li>
            </ol> 
            <p></p> 
            <h3><a name="Secondo_Esonero"></a> Secondo Esonero </h3> 
            <p></p> L'esonero si terrà 
            <strong>martedì 8 giugno ore 12.00, aula alfa</strong>. 
            <p></p> 
            <h2><a name="Programma"></a> Programma </h2> 
            <p></p> 
            <ol> 
             <li> Un modello di computazione distibuita; stati globali e ordinamento degli eventi; </li> 
             <li> Logical time; </li> 
             <li> Snapshot distribuito; </li> 
             <li> Mutua esclusione; </li> 
             <li> Detection of non-stable predicates; </li> 
             <li> Distributed commit; </li> 
             <li> Consensus; </li> 
             <li> Failure detectors; </li>
            </ol> 
            <p></p> 
            <a href="/twiki/view/SD/Diario0910" class="twikiLink">Diario delle lezioni, articoli e dispense.</a> 
            <p></p> 
            <h3><a name="Libro"></a> Libro </h3> 
            <p></p> Ajay D. Kshemkalyani and Mukesh Singhal. 
            <em>Distributed Computing: Principles, Algorithms, and Systems</em>, Cambridge University Press, 2008. ISBN-13 978-0-521-87634-6. 
            <p></p> 
            <h2><a name="Appelli"></a> Appelli </h2> 
            <p></p> Il modo privilegiato per passare questo esame è grazie ai due esoneri, quello intermedio e quello finale. Faremo il secondo esonero nella prima parte di giugno. Chi passa gli esoneri non deve fare l'orale. Tutti gli altri appelli dell'anno avranno l'orale obbligatorio. 
            <p></p> 
            <h2><a name="Risultati_2009_10"></a> Risultati 2009-10 </h2> 
            <p></p> 
            <a class="twikiLink" href="/twiki/view/SD/RisultatiProvaFinali0910">Risultati finali 2009-10.</a> 
            <p></p> Appuntamento per la verbalizzazione (chi vuole fare l'orale in questo appuntamento lo può fare ma me lo deve dire prima) il giorno 
            <strong>7 luglio, ore 15, aula Seminari</strong>. Chi 
            <strong>deve</strong> verbalizzare prima per motivi importanti (tipo laurea a luglio) può prendere un appuntamento separato. Considerate che nelle due settimane dal 21 giugno al 2 luglio sono presente solo venerdì 25 giugno. 
            <font color="red">Attenzione!</font> Dalle 14 alle 15.30 il Direttore ha convocato una riunione urgente e importante dei docenti del Dipartimento in aula Seminari! Di conseguenza il nostro appuntamento è spostato alle 15.30 (anche se la mia esperienza mi dice che ci potrebbe essere un piccolo ritardo ulteriore.) 
            <p></p> 
            <h2><a name="FAQ"></a> FAQ </h2> 
            <p></p> 
            <em>Ehi! Non trovo Sistemi distribuiti nell'orario di quest'anno! Non si fa? Dov'è finito Alessandro Mei?</em> 
            <p></p> 
            <strong>Risposta:</strong> No, non si fa, ma si farà regolarmente il prossimo anno. Alessandro Mei, nel frattempo, è professore ospite alla University of California San Diego. Ma con il cuore è sempre tra voi e risponde regolarmente (beh, quasi regolarmente) alle email al vecchio indirizzo. 
            <p></p> 
            <em>Dove posso trovare il libro di testo?</em> 
            <p></p> 
            <strong>Risposta:</strong> In biblioteca c'e' una copia del libro. L'ho tenuto in prestito per un po' (circa due anni), ma adesso è di nuovo in biblioteca. 
            <p></p> 
            <em>Sono uno studente lavoratore, posso ugualmente fare gli esoneri?</em> 
            <p></p> 
            <strong>Risposta:</strong> Si. Mi raccomando, si tenga aggiornato su questo sito soprattutto con il diario delle lezioni. 
            <p></p> 
            <em>Non ho fatto il primo esonero. (Oppure, ho preso un voto molto insufficiente al primo esonero.) Posso fare il secondo?</em> 
            <p></p> 
            <strong>Risposta:</strong> Si. Possiamo recuperare il primo esonero con un orale.

var twTopMenuBarCloseTimer = null;
var twTopMenuBarTimerMsec = 1000;
function twToggleTopMenuBar( isHover ) {
  if( 'off' == 'on' || 'off' == '1' ) {
    return;
  }
  var displayState = 'none';
  if( $(window).scrollTop()<22 || isHover>0 ) {
     displayState = 'block';
     if(twTopMenuBarCloseTimer) { window.clearTimeout(twTopMenuBarCloseTimer); twTopMenuBarCloseTimer = null; } 
  } else {
    if(!twTopMenuBarCloseTimer && isHover>=0 ) {
      twTopMenuBarCloseTimer = window.setTimeout(twCloseTopMenuBar, twTopMenuBarTimerMsec);
      return;
    }
  }
  if( $('#topmenu-bar').css('display') != displayState ) {
    $('#topmenu-bar').css('display', displayState);
  }
};
function twCloseTopMenuBar() {
  twToggleTopMenuBar(-1);
}
$(document).ready(function () {
  twToggleTopMenuBar(0);
});
$(window).scroll(function () {  
  twToggleTopMenuBar(0);
});
$('#topmenu-start').hover(
  function () { twToggleTopMenuBar(1); }, 
  function () { twToggleTopMenuBar(0); }
);
</script>
